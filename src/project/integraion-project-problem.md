# 如何做多个业务系统中前端的集成?

## 信息孤岛
服务型公司 -> 背景是给客户做交付,一般两种可能
* 从无到有新做 (直接帮客户信息化)
* 和现有系统集成 (已有信息化,需要迭代或补充)

如果你是第二种, 那么你就会遇到信息孤岛的问题

## 集成要面临的选择
* 系统间的登录态互认 - OA SSO COOKIE SESSION
* 系统间的协议 - 接口调用
* 技术栈统一性问题 - 微前端 iframe(前几年常见) compact
* 数据源打通 (系统协作)
* 导入新数据 -> 逐步替代新系统(老系统停止产生新数据/新系统共存一段后老系统下线)
* 也可能完全切换上线(上线后老系统下线)

## 今天聊几个前端层面可能遇到的问题
* 登录态问题
* 权限问题
* 数据问题
* 页面集成
* 风格统一
